## 💡 核心能力

- **需求分析**：從用戶描述中提取核心需求和痛點
- **結構化輸出**：生成符合業界標準的 PRD 格式
- **場景化設計**：設計符合實際使用場景的產品功能
- **迭代優化**：根據反饋持續改進需求文件

## 🎯 適用場景

- 想要將想法轉化為產品需求
- 需要為新產品或功能撰寫 PRD
- 需要將模糊需求轉化為清晰規格
- 需要為開發團隊提供明確指引

## 📝 使用範例

### 範例 1：建立股票 LINE Bot PRD

**用戶輸入**：
> 我想做一個可以查詢台股即時價格和 AI 分析的 LINE Bot

**技能輸出**：
```markdown
# 產品需求文件：股票助手 LINE Bot

## 1. 產品概述
建立一個 LINE Bot，讓使用者透過 LINE 即時查詢台灣股票價格、
獲得 AI 分析建議，並管理個人追蹤清單。

## 2. 目標用戶
- 台灣股市散戶投資人
- 想要快速查看股價的上班族
- 對股票分析有興趣的初學者

## 3. 核心功能
- 即時股價查詢（twstock）
- AI 股票分析（Gemini）
- 個人追蹤清單（SQLite）

## 4. 技術架構
- 後端：FastAPI + Python
- LINE SDK：line-bot-sdk v3
- AI：Google Gemini API
- 資料庫：SQLite

## 5. 成功指標
- Bot 可正常接收並回覆訊息
- 股價查詢結果準確
- AI 分析回覆具參考價值
```

## 📊 輸出格式

```markdown
# 產品需求文件：[產品名稱]

## 1. 產品概述
[簡短描述產品目標和願景]

## 2. 目標用戶
[描述目標用戶群體和他們的特徵]

## 3. 核心功能
[列出核心功能列表]

### [功能名稱]
- [功能描述]
- [使用場景]
- [預期結果]

## 4. 非功能需求
[性能、安全、可用性等要求]

## 5. 技術架構
[技術棧和系統架構]

## 6. 成功指標
[如何衡量產品成功]

## 7. 實施路線圖
[開發階段和里程碑]
```

## 🛠️ 使用指南

### 最佳實踐

1. **提供清晰的背景信息**：說明產品的目標和願景
2. **明確需求範圍**：說明哪些功能是必須的、可選的
3. **提供具體細節**：描述每個功能的具體行為
4. **迭代優化**：根據反饋修改需求
